## Design Context

### Users
Cashiers, supervisors, and admins operating a point-of-sale workflow in a browser. They need to browse items, build a basket, submit sales, inspect dashboard insights, and manage hidden admin tools quickly without losing their place.

### Brand Personality
Playful, polished, and approachable. The interface should feel like a candy-soft terminal: friendly rather than sterile, confident rather than flashy, and practical before decorative.

### Aesthetic Direction
Soft pastel POS terminal with rounded surfaces, layered depth, and a light-first theme that can switch to dark mode. Avoid neon tron styling, hard-glow surfaces, and overly technical visual language. Favor rounded containers, airy spacing, tactile controls, and a calm but lively palette.

### Design Principles
- Keep hierarchy obvious with space, weight, and section separation before color.
- Use a consistent spacing scale with stronger breathing room between major regions and tighter grouping within controls.
- Preserve responsive behavior so checkout, dashboard, and admin layouts adapt without collapsing into cramped cards.
- Make interactive elements feel tactile and immediate with clear hover, focus, and active states.
- Keep the theme switch visible and stable so the color system feels intentional in both light and dark modes.
